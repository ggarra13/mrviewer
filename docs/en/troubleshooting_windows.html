Troubleshooting

Windows issues

mrViewer does not show its command-line options.

Whenever you try to start up image viewer, it does not log information to the console.

Answer:
This is a problem with windows. To see the options type:
mrViewer -h

The options will appear a new console window. To see the rest of the messages, go to
Windows->Logs

mrViewer does not start giving a ms-api-core* message or similar

Answer:
This can be several problems. You may be mixing x86 and x64 DLLs. You can check them with the Dependency Walker utility freely available on the internet. Another possibility is that you may have a version of mrViewer compiled with the Win10 SDK which makes it compatible only with Win8 / Win8.1 / Win10. By mistake several old versions of mrViewer were using that SDK.
