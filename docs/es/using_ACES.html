Using ACES (Clip Files)

mrViewer es largamente compatible con ACES (Academy Color Encoding System) para la corrección de color y el manejo del mismo.

mrViewer lee archivos ACESClip xml  que describen las diferentes transformaciones de CTL aplicadas a una imagen.  Tambien contiene ASC-CDL correcciones de color que son emuladas como Look Mod Transforms.

mrViewer puede tener multiple transformaciones de CTL activas al mismo tiempo.  Una transformación está atada a la vista de la ventana principal y representa el monitor, video o film stock.  Esta transformación dentro de la convención de nombres dentro de CTL es llamada la Transformación de Aparato de Salida o Output Device Transform (ODT).  Este transform se setea en las ventana principal de preferencias, en la sección de CTL.  La mayoría de projectos requerirá que los usuarios seteen la ODT una vez y no la cambien.

mrViewer viene con varias ODTs básicas para emular los monitores comunes.

Una ODT que intenta emular film requerirá frecuentemente una más completa tabla de lut de 2D o 3D para intentar emular la inversa de la Transformación de Emulación de Impresión en Fílmico o Print Film Emulation Transform (PFET).  La PFET es particular a cada film stock y laboratorio de revelado de film y generarla requiere una evaluación cuidadosa del film stock y sus condiciones de revelado.  Para más información acerca de crear la inversa PFET, usted debería consultar con su laboratorio de revelado.

Una segunda transformación disponible es la Transformación de Rendering o Rendering Transform o RT para abreviar.   Esta transformación es unica a cada imagen y proyecto y representa la transformación necesaria para llevar los valores de pixel RGB guardados en una imagen a una agradable representación interna de CIE XYZ (OCES) para el proyecto.  mrViewer viene con una basica Rendering Transform que fue dada por AMPAS para proveer una Transformación de Rendering de Referencia o Reference Rendering Transform que debería convertirse en el estandar para la mayoria de las produciones.  Si tu proyecto requiere algun effect especial de film (como film lavanda), una transformación de rendering especial podría tener que ser usada.

Una tercera transformación disponible son las Transformaciones de Modificación de Look o Look Modification Transform (LMT).  Esta transformación esta pensada principalmente para cámaras para llevar su imagenes de input y corregir su color basado en ciertos parametros como se hace en el set y esto es particular a cada modelo de cámara. Usted puede tener un número ilimitado de LMTs.

Finalmente, está una Transformación de Aparato de Entrada o Input Device Transform (IDT) que es una transformación usada para llevar los datos en crudo de la cámara a un propicio espacio de trabajo de ACES. mrViewer viene con una pequeña lista de IDTs para algunas cámaras de fílmico populares.  Si su camara no está listada, contacte a su manufacturador de su camera.

Todas estas transformaciones pueden ser almacenadas en un archivo de Clip de Metadata de ACES.  mrViewer carga equivalent archivos de Clip de Metadata de ACES si se encuentran.  Por ejemplo, para una secuencia llamada nature.%04d.exr, el archivo clip llamado ACESclip.nature.xml sería buscado y cargado si está presente.  Una vez que el archivo de clip es cargado puede presionar el botón de LUT en la barra de herramientas principal y esperar que se cocine la transformación (que debería llevar unos segundos).
